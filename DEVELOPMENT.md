# 开发文档 - One API 渠道批量更新工具

本文档记录了该工具的开发状态、架构和后续步骤。

## 项目目标

提供一个命令行工具，用于批量更新 One API 实例中的渠道配置。支持通过 JSON 文件定义筛选条件和更新规则，并能处理不同版本的 One API (newapi, voapi)。增加撤销（Undo）功能以提高操作安全性。

## 当前状态 (截至 2025-04-19)

1.  **代码重构:**
    *   核心的配置加载、渠道过滤、更新数据准备等通用逻辑已提取到 `channel_tool_base.py` 中的抽象基类 `ChannelToolBase`。
    *   `newapi_channel_tool.py` 和 `voapi_channel_tool.py` 现在继承自 `ChannelToolBase`，并实现了各自 API 版本特定的方法（如 `get_all_channels`, `update_channel_api`, `get_channel_details`）。这使得代码结构更清晰，更易于维护和扩展。

2.  **主入口 (`main_tool.py`):**
    *   作为脚本的主要入口点，负责解析命令行参数 (`argparse`)。
    *   提供交互式提示，用于选择连接配置和 API 类型（如果未通过命令行参数指定）。
    *   统一处理日志记录的设置，现在日志会保存到 `logs/` 目录下，并带有时间戳。
    *   协调整个更新或撤销流程，根据用户选择或参数调用相应的功能。

3.  **撤销 (Undo) 功能:**
    *   已在 `main_tool.py` 中添加了 `--undo` 参数和基本框架。
    *   实现了 `save_undo_data` 函数：在执行实际更新 **之前**，此函数会：
        *   获取所有渠道。
        *   根据 `update_config.json` 中的 `filters` 筛选出将要被更新的渠道。
        *   通过 API 获取这些匹配渠道的 **当前完整状态**。
        *   将这些原始状态数据保存到一个位于 `undo_data/` 目录下的、带有时间戳和配置标识的 JSON 文件中。
    *   实现了 `perform_undo` 函数：此函数会：
        *   读取 `undo_data/` 目录下最新的撤销文件。
        *   使用文件中保存的原始渠道数据，调用相应的 `update_channel_api` 方法，将渠道恢复到保存时的状态。
    *   实现了 `find_latest_undo_file` 函数，用于定位最新的可用撤销文件。

4.  **日志改进:**
    *   日志现在会输出到控制台和文件。
    *   日志文件默认保存在 `logs/` 目录下，并包含时间戳 (`channel_updater_YYYY-MM-DD-HHMMSSmmm.log`)。
    *   可以通过 `--log-file` 参数指定日志文件路径或目录，或通过传递空字符串 `""` 或 `"none"` 来禁用文件日志。
    *   可以通过 `--log-level` 参数控制日志级别。

## 后续步骤

1.  **测试更新功能:**
    *   需要针对 `newapi` 和 `voapi` 两种类型，使用不同的 `connection_config.json` 和 `update_config.json` 组合进行全面的更新测试。
    *   验证筛选逻辑是否按预期工作。
    *   验证各种更新字段（字符串、数字、布尔、JSON 对象、列表）是否能正确应用。
    *   检查 Dry Run (`--dry-run`) 模式是否能正确显示计划的更改而不实际执行。

2.  **测试撤销功能:**
    *   **验证 `save_undo_data`:** 运行一次更新操作，检查 `undo_data/` 目录下是否生成了正确的撤销文件，以及文件内容是否包含了预期渠道的原始状态。
    *   **验证 `perform_undo`:**
        *   运行一次更新。
        *   手动修改 One API 中的某个已更新渠道（或运行另一次不同的更新）。
        *   运行带有 `--undo` 参数的命令，选择相同的连接配置和 API 类型。
        *   检查之前被修改的渠道是否已成功恢复到撤销文件中记录的状态。
    *   测试撤销文件不匹配时的警告和确认流程。

3.  **代码和逻辑优化:**
    *   根据测试结果，优化错误处理逻辑，提供更清晰的错误信息。
    *   改进日志记录，确保关键信息都被记录下来，同时避免冗余信息。
    *   优化用户交互提示和最终的执行结果反馈。

4.  **VO API 适配确认:**
    *   当前 `voapi_channel_tool.py` 中的 `update_channel_api` 和 `get_channel_details` 实现很大程度上是基于与 `newapi` 相似的假设。需要实际测试或查阅 VO API 文档，确认其更新和获取详情的 API 端点、请求方法（PUT/POST?）、认证方式（Bearer Token 是否足够？）以及请求/响应体结构是否与当前实现一致。如有差异，需进行适配。

5.  **更新 README:**
    *   在 `README.md` 中补充详细的使用说明，特别是关于新的 `--undo` 功能、日志文件行为以及配置文件（`filters` 和 `updates`）的详细格式说明和示例。
